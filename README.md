# GPAGO 자동 분석 — 브라우저 확장 (수동 캡처 모드)

네이버 쇼핑 검색 결과를 GPAGO로 전송하여 자동 분석합니다. **페이지를 자동으로 조작하지 않으며**, 사용자가 페이지에서 자연스럽게 행동(정렬·페이지 변경 등)할 때만 데이터를 가로채는 안전한 방식입니다.

## ⚠️ 차단 방지 안전 모드

이전 버전(v1.x)은 페이지를 자동으로 클릭하거나 URL을 변경했는데, 네이버 anti-bot 시스템이 이를 탐지하여 IP 차단이 발생할 수 있었습니다. 현재 버전은:

- ✅ 페이지 동작을 일체 조작하지 않음 (자동 클릭·URL 변경 없음)
- ✅ 추가 API 요청 보내지 않음
- ✅ 페이지가 자체적으로 발생시키는 fetch 응답만 관찰
- ✅ 사용자가 보고 있는 페이지의 데이터만 GPAGO로 옮김

= 단순 "복사-붙여넣기" 자동화 수준 → 차단 위험 거의 없음

## 설치 방법

1. `chrome://extensions` (Edge는 `edge://extensions`, Whale은 `whale://extensions`)
2. 우측 상단 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램을 로드합니다** 클릭
4. `gpago-extension` 폴더 선택
5. 툴바의 퍼즐 아이콘 → 📌 핀으로 고정

## 사용 방법

### 1. 일반 사용 패턴
1. 네이버 쇼핑에서 키워드 검색
2. **80개씩 보기 선택** 또는 **정렬 변경** 등 한 번의 페이지 조작 (자연스러운 사용)
3. → 확장이 백그라운드에서 자동으로 데이터 캡처
4. 확장 아이콘 클릭 → GPAGO 탭이 열리며 자동 분석

### 2. 캡처가 없을 때
- 키워드 검색 후 페이지 조작 없이 바로 확장 클릭
- → 상단에 안내 배너 표시: "페이지에서 한 번만 조작해 주세요"
- 사용자가 정렬·페이지 변경 등 하면 → 자동으로 GPAGO 분석 시작
- 1.5분 안에 조작 안 하면 → 시간 초과 알림

## 동작 원리

| 단계 | 동작 | 안전성 |
|---|---|---|
| 1 | 페이지 로드 시 inject 스크립트가 fetch/XHR을 **관찰만** | 관찰은 정상 사용과 구분 불가 |
| 2 | 사용자가 페이지에서 행동 → 페이지가 스스로 fetch 호출 | 사용자 직접 행동 |
| 3 | inject 스크립트가 응답을 chrome.storage에 저장 | 페이지 외부에 데이터 옮김 |
| 4 | 확장 아이콘 클릭 → GPAGO 탭으로 데이터 전달 | 단순 데이터 이동 |

**중요**: 확장이 페이지를 클릭하거나 URL을 변경하지 않습니다. 모든 페이지 조작은 사용자가 직접 합니다.

## 차단당했을 때

만약 네이버 "쇼핑 서비스 접속이 일시적으로 제한되었습니다" 페이지가 뜨면:

1. **확장 비활성화** (chrome://extensions에서 OFF)
2. **10~30분 대기** (보통 일시 차단)
3. 그 사이 네이버 쇼핑을 정상 사용 (확장 없이)
4. 차단 풀린 후 확장 다시 활성화

## 파일 구조

```
gpago-extension/
├── manifest.json       (확장 정의)
├── background.js       (아이콘 클릭 처리, 캡처 데이터 전달)
├── content-naver.js    (네이버 페이지에 inject 스크립트 주입)
├── inject-naver.js     (페이지 컨텍스트에서 데이터 관찰만 — passive)
├── content-gpago.js    (GPAGO 페이지에 데이터 전달)
└── README.md           (이 문서)
```

## 문제 해결

### 캡처가 안 됨
- 페이지에서 정렬·페이지 크기를 한 번이라도 바꾸셨나요?
- F12 → Console 탭에서 `[GPAGO] captured from xhr:...` 메시지가 보이는지 확인

### "캡쳐 대기 중" 배너만 계속 뜸
- 페이지를 한 번 조작해 주세요 (80개씩 보기 선택이 가장 효과적)
- 또는 가격 정렬 → 다시 네이버 랭킹순으로 변경

### 그래도 안 됨
- 기존 수동 방법 사용 가능:
  1. F12 → Network 탭 → "search" 또는 "all" 필터
  2. 응답 중 products 배열이 있는 JSON 찾기
  3. Response 복사 → GPAGO의 JSON 붙여넣기 영역에 붙여넣고 분석 실행
